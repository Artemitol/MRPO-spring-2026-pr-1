# Главные требования проекта (сводка)

## Роли и права

- Гость: просмотр списка товаров без поиска, фильтрации, сортировки.
- Авторизованный клиент: просмотр списка товаров без поиска, фильтрации, сортировки.
- Менеджер: просмотр товаров с поиском/фильтром/сортировкой, просмотр заказов.
- Администратор: полный CRUD товаров и заказов, плюс функции менеджера.

## База данных (модуль 1)

- СУБД: PostgreSQL.
- Нормализация: 3НФ.
- Обязательны PK/FK, ссылочная целостность, согласованное именование.
- Ограничения: неотрицательная цена и остаток, валидная скидка, даты заказа, запрет удаления товара из заказа.
- Артефакты: SQL-скрипт БД, ER-диаграмма PDF, подготовка импортируемых данных.

## Приложение (модуль 2)

- Стартовый экран: окно входа.
- Доступен вход по логину/паролю или гостевой вход.
- После входа отображается ФИО пользователя (правый верхний угол).
- Должен быть доступен выход на главный экран.
- Список товаров берется из БД.
- Внешний вид по style guide: Times New Roman, цвета #FFFFFF, #7FFF00, #00FA9A.

## UI и логика (модуль 3)

- Последовательная навигация между экранами, включая возврат назад.
- Заголовок должен быть у каждого экрана.
- Обработка ошибок и предупреждений с информативными сообщениями.
- Поиск по текстовым полям товара (realtime).
- Фильтр по поставщику (первый элемент: «Все поставщики»).
- Сортировка по количеству на складе (asc/desc).
- Поиск + фильтр + сортировка работают совместно.
- CRUD товаров только для админа.
- Форма товара: поля по ТЗ, валидация числовых значений.
- ID: при добавлении не вводится вручную; при редактировании read-only.
- Изображение: хранится путь в БД, есть заглушка, ограничение по размеру 300x200.
- После add/edit/delete список обновляется.

## Заказы (модуль 4, бонус)

- Экран заказов у менеджера и админа.
- CRUD заказов только у админа, просмотр у менеджера.
- Поля заказа: артикул, статус, адрес, дата заказа, дата выдачи.
- После add/edit/delete список обновляется.

## Визуальные правила для списка товаров

- Если скидка > 15%: фон #2E8B57.
- Если товар уценен: старая цена зачеркнута красным, рядом итоговая цена черным.
- Если товара нет на складе: строка выделяется голубым.
- На главной форме должен быть логотип и заголовок.
- Иконка приложения должна быть установлена, если платформа позволяет.
